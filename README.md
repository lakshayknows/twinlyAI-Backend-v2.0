# TwinlyAI Backend

This is the backend service for the TwinlyAI SaaS application. It is built using **FastAPI** and **MongoDB**, heavily utilizing **LangChain** and **Groq** to power AI bot interactions, RAG (Retrieval-Augmented Generation), and text-to-speech features. 

The backend also includes support for real-time voice and video using **Agora**, and is containerized via **Docker** for easy deployment, specifically optimized for platforms like **Hugging Face Spaces**.

## 🚀 Tech Stack

- **Framework**: [FastAPI](https://fastapi.tiangolo.com/) with Uvicorn
- **Database**: [MongoDB](https://www.mongodb.com/) (Motor/PyMongo)
- **AI & LLM**: [LangChain](https://www.langchain.com/), [Groq API](https://groq.com/)
- **Embeddings & Vector Search**: [Sentence Transformers](https://huggingface.co/sentence-transformers) (`BAAI/bge-small-en-v1.5`), FAISS, Qdrant
- **Authentication**: JWT (python-jose), Bcrypt, OAuth, Authlib
- **Document Processing**: `pdfplumber`, `python-docx`
- **Text-to-Speech**: `edge-tts`
- **Real-time Communication**: Agora SDK

## 🔌 API Endpoints (v1)

The API is versioned (`/api/v1`) and module-driven:

- **`/auth`**: Authentication endpoints (Registration, Login, JWT issuing).
- **`/oauth`**: Social login integration (OAuth).
- **`/bots`**: Creation, management, and querying of AI Twin bots.
- **`/api-keys`**: Management of developer/user API keys.
- **`/users`**: User profile and account management.
- **`/recruiter`**: Tools for recruiters (managing candidates, voice interviews, assessing profiles).
- **`/agora`**: Token generation and webhook handling for Agora real-time voice/video functionality.
- **`/webhooks`**: Reception of external event webhooks.

*A full interactive API documentation automatically generated by FastAPI is available at `/docs` (Swagger UI) or `/redoc` when the server is running.*

## ⚙️ Environment Variables

Ensure you have a `.env` file in the root of the `backend/` directory. Required variables typically include:

```ini
ENV=dev
SESSION_SECRET_KEY=your_session_secret_here
GROQ_API_KEY=your_groq_api_key
MONGODB_URL=mongodb+srv://...
JWT_SECRET=your_jwt_signature_secret
```

## 🛠️ Running Locally

### 1. Set up a virtual environment

```bash
python -m venv venv
# On Windows
venv\Scripts\activate
# On macOS/Linux
source venv/bin/activate
```

### 2. Install dependencies

```bash
pip install -r requirements.txt
```

### 3. Run the application

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
The API should now be accessible at `http://localhost:8000`.

## 🐳 Docker Deployment

This repository comes with a multi-stage Dockerfile optimized to pre-fetch Hugging Face embedding models (e.g., `BAAI/bge-small-en-v1.5`) during the build stage to ensure fast container start times and reliable deployment on Hugging Face Spaces.

### Build the Image
```bash
docker build -t twinlyai-backend .
```

### Run the Container
```bash
docker run -p 7860:7860 --env-file .env twinlyai-backend
```
*Note: The Docker container defaults to exposing port 7860.*
